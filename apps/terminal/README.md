# apps/terminal

The HypeTerminal web app. React 19 + TanStack Start (SSR) + Vite 7.

Composes `@hypeterminal/hl-react` (data/signing) and `@hypeterminal/ui` (design system) into the product. See the [root README](../../README.md) for how it fits into the monorepo.

## Stack

| Layer | Choice | Version |
|---|---|---|
| Framework | TanStack Start (SSR meta-framework on Vite + Nitro) | 1.132 |
| Bundler | Vite | 7.3 |
| React | React + React Compiler (babel plugin) | 19.2 |
| Server state | TanStack Query | 5.90 |
| Client state | Zustand (+ `persist`) | 5.0 |
| Styling | Tailwind v4 + `@hypeterminal/ui` tokens | 4.0 |
| Wallets | wagmi + viem | 3.1 / 2.42 |
| Charting | klinecharts (+ optional TradingView via `types/charting_library`) | 9.8 |
| Bridging | `@lifi/sdk` | 3.16 |
| Numbers | `big.js` (via `lib/trade/numbers`) | 7.0 |
| Validation | Zod | 4.3 |
| i18n | Lingui (6 locales) | 5.7 |
| Tables / virtualization | `@tanstack/react-table` + `@tanstack/react-virtual` | — |
| Toasts / drawer | `sonner`, `vaul` | — |
| Tests | Vitest (+ WebSocket soak harness) | 4.0 |

Vite config is in `vite.config.ts`. Notable bits:

- `tanstackStart` + `nitroV2Plugin` for SSR
- `@vitejs/plugin-react` wired with `babel-plugin-react-compiler`
- `@lingui/vite-plugin` + `@lingui/babel-plugin-lingui-macro` for i18n
- `@tailwindcss/vite` for Tailwind v4
- Manual chunking splits vendors: `vendor-radix`, `vendor-tanstack`, `vendor-charts`, `vendor-web3`, `vendor-klinecharts`
- `rollup-plugin-visualizer` for bundle analysis

### Providers

`src/providers/root.tsx` composes:

- `QueryClientProvider` (TanStack Query)
- `I18nProvider` (Lingui)
- `WagmiProvider` + connectors (`config/wagmi.ts`)
- `HyperliquidProvider` from `hl-react` — configured with `env`, `builderConfig`, `agentName`
- `MarketsInfoProvider` — market metadata
- `ExchangeScopeProvider` — perp vs spot scope

## Styling

`src/styles.css` is the entry. It imports:

```css
@import "tailwindcss";
@import "tw-animate-css";
@import "@fontsource-variable/inter";
@import "@hypeterminal/ui/globals.css";   /* design-system tokens */
/* app-level tokens + dark-mode variant follow */
```

App-level tokens extend the `@hypeterminal/ui` base. `@source "../../../packages/ui/src"` tells Tailwind to scan the UI package for class usage.

See `.claude/rules/design-tokens.md` for the token taxonomy (`bg-bg-sunken`, `text-text-strong`, `bg-market-up`, etc.) — **never use hex literals**.

## Testing

`src/lib/tests/` holds all vitest suites:

- Unit: `format.test.ts`, `errors.test.ts`, `orders.test.ts`, `trades.test.ts`, `seo.test.ts`, `ring-buffer.test.ts`
- Integration: `hyperliquid-hooks.test.ts`, `websocket-payload-guard.test.ts`
- Perf/soak: `websocket-soak.test.ts`, `websocket-store-benchmark.test.ts`

```bash
pnpm test                   # full vitest run
pnpm perf:ws-soak:30m       # 30-min WS reliability soak
pnpm perf:ws-soak:60m       # 60-min variant
pnpm perf:bundle            # bundle size analysis
```

Soak knobs: `WS_SOAK_RUN`, `WS_SOAK_DURATION_MS`, `WS_SOAK_SAMPLE_MS`, `WS_SOAK_OP_MS`.

## i18n

`lingui.config.ts` — source locale `en`, catalogs at `src/locales/{locale}/messages.po`. Locales: `en`, `zh`, `hi`, `es`, `fr`, `ar`.

`src/lib/i18n.ts` exposes `locales`, `localeList`, `numberFormatLocales`, and `dynamicActivate(locale)` (lazy-loads catalog via `import.meta.glob`). `en` is loaded synchronously on startup for SSR.

In components:

```tsx
import { t } from "@lingui/core/macro";
import { Trans } from "@lingui/react/macro";
```

## Environment

Env vars are read via `import.meta.env` in `src/lib/network.ts` and `src/config/wagmi.ts` — mostly `VITE_*` flags for testnet/mainnet and chain selection.

## Conventions

See the root [README](../../README.md#conventions) and `.claude/rules/`. In this app specifically:

- Trade math goes in `lib/trade/`, not components.
- Components use `@hypeterminal/ui` primitives; app-specific compositions go in `components/ui/`.
- SSR: never touch `window`/`document` at module top-level. Use `ClientOnly` from `@tanstack/react-router` for client-only trees.
- No manual `useMemo`/`useCallback` for performance — the React Compiler handles it.
